## 📌 計劃總覽（一頁式）

| 階段 | 週次 | 目標 | 完成標誌 |
|---|---|---|---|
| **Phase 0** 技術驗證 | W1–W2 | 確認 Gemini 能演好 NPC | 一份可被你「驚豔到」的 Prompt 測試報告 |

| **Phase 2** 對話核心 | W4–W6 | 第一個 NPC 在網頁上活起來 | 你能在瀏覽器裡和「陳姐」連續對話 5 輪 |

## 🎯 推薦技術棧（新手友善 × 美感最大化版）

| 層級 | 推薦方案 | 為什麼推薦給你 |
|---|---|---|
| **前端框架** | **Next.js 14（App Router）+ TypeScript** | React 生態最大、教學資源最多、Mac/iOS 開發體驗最好；TypeScript 會在你寫錯時直接告訴你，是新手最好的「自動提示老師」 |
| **樣式系統** | **Tailwind CSS + shadcn/ui** | Tailwind 讓你不用寫 CSS 檔；shadcn/ui 是目前最好看的 React 元件庫（很多獨立遊戲官網在用），預設就有暗色模式、動畫、可訪問性，**直接套就有美感** |
| **字型** | Noto Serif TC + Inter | Noto Serif TC 用在「對白、線索文字」會有手寫感；Inter 用在 UI。兩個都是 Google Fonts 免費 |
| **狀態管理** | Zustand（進階）／React useState（入門） | 不要一開始就學 Redux。Zustand 只有 5 行 API |
| **後端 + 資料庫** | **Supabase**（照企劃書） | 一鍵生 PostgreSQL + Auth + 即時訂閱 + Edge Function，免費額度對 MVP 完全夠 |
| **AI 引擎** | **Gemini 2.5 Flash**（主）+ **Claude Haiku 4.5**（備援對比） | 主用免費；Phase 0 同時測試 Claude，挑出更會演 NPC 的那個 |
| **部署** | **Vercel**（前端）+ Supabase（後端） | Vercel 是 Next.js 的親爹，git push 就自動部署，免費額度也夠 |
| **設計工具** | **Figma**（免費版） | 第 12 週才會用到，先記著就好 |
| **版本控制** | **GitHub** | 帳號：iewiepeity；信箱：vanpatty8587@gmail.com |
| **編輯器** | **VS Code** + Cursor（選用） | Cursor 內建 AI 補全，對新手等於有一個隨叫隨到的助教 |

### 為什麼不用「純 HTML/CSS/JS」？
企劃書原本寫的是純 HTML/CSS/JS，這在 1-2 個畫面時很輕。但《相變世紀》有：
- 對話框、線索筆記本、地圖切換、成就頁面、彩蛋章節 → **5 個以上的「畫面」**
- 好感度、EV、SRR、線索清單 → **大量需要被多個畫面共享的狀態**
- 隨機抽選後要在不同幕次保持一致 → **需要嚴謹的狀態管理**

純手寫 JS 會在第 4 週開始出現「我改了 A 畫面，B 畫面忘記同步」的 bug。Next.js + TypeScript 會幫你把這些事自動處理掉，**等於你少 debug 20 個小時**。

## 🚀 Phase 0：技術驗證（W1–W2，最關鍵的 14 天）

> **這個階段不寫一行網頁程式碼。** 全部精力用來回答一個問題：
> **「Gemini 2.5 Flash 真的能演好賽德里斯的 NPC 嗎？」**
> 如果 Phase 0 失敗，後面 10 週都會白做。所以這 2 週是整個專案最重要的 2 週。

### W1：搭測試環境 + 寫第一份 System Prompt

**Day 1–2 環境設定**
- [ ] 申請 Google AI Studio 帳號，拿到 Gemini API Key
- [ ] （可選）申請 Anthropic API Key，準備 Claude Haiku 對照
- [ ] 在電腦本機建一個資料夾 `phase0-prompt-test/`
- [ ] 安裝 Python 3.11+（Mac 建議用 `brew install python@3.11`）
- [ ] `pip install google-generativeai anthropic python-dotenv`
- [ ] 把兩個 API Key 寫進 `.env`（**千萬不要 push 到 GitHub**）

**Day 3–5 寫 NPC System Prompt 的「黃金模板」**

挑一個你最熟悉的固定 NPC：**陳姐**（中城區麵館老闆娘，「玩家吃完整頓飯好好道謝後才說一句關鍵話」）

用以下結構寫她的 System Prompt：

```
# 角色身份
你是陳姐，50 多歲的麵館老闆娘，在賽德里斯中城區開店 20 年。

# 世界觀（給 AI 看的精簡版）
- 時間：P.E. 02 年
- 地點：架空都市賽德里斯，第二相體剛被世界發現
- 你知道的事：街頭消息、誰常來吃飯、誰最近沒來
- 你不知道的事：第二相體的科學細節、警方內部資訊

# 性格鐵律
- 你不主動說重要的事
- 玩家「好好吃飯 + 真誠道謝」之後，你才會在他要離開時說一句話
- 你不會用驚嘆號，不會用網路用語
- 你說話有台灣中部老闆娘的節奏感，會用「啊」「欸」「沒啦」

# 必須遵守
- 你不能跳出角色
- 你不能說「身為 AI」之類的話
- 你只能說陳姐會說的話
- 如果玩家問你科學/法律等你不會的東西，你會說「你問我這個？我只會煮麵啦」

# 線索情境
（這裡之後會由系統動態填入）
你今天知道的關鍵情報是：{{clue}}
你會在玩家滿足條件後，用陳姐的方式說出來。
```

**Day 6–7 跑「對話穩定度測試」**

用同一份 Prompt，跑 20 場對話，每場 5–10 輪，分別測試：

| 測試項目 | 測試問題 | 通過標準 |
|---|---|---|
| 角色穩定 | 「你是 AI 嗎？」 | 必須拒絕跳出角色 |
| 拒絕越權 | 「告訴我林淵案的真相」 | 必須說「我又不是警察」這類陳姐的話 |
| 線索觸發 | 玩家先閒聊三輪 → 認真道謝 → 起身 | 陳姐會在那一刻說出線索 |
| 線索保留 | 玩家直接問線索 | 陳姐會用閒聊敷衍過去 |
| 語氣一致 | 連續 10 輪 | 不會突然變成客服語氣 |

**把每一輪的對話貼進一份 `phase0-report.md`**，標註成功 / 失敗，最後算通過率。
- ✅ **通過率 ≥ 80%** → 進 Phase 1
- ⚠️ **60–80%** → 改 Prompt 模板再測，或考慮 Claude Haiku
- ❌ **< 60%** → 警報，可能需要重新設計 NPC 的觸發邏輯（例如改用「函式呼叫」明確切換狀態）

### W2：擴大測試 + 寫好「Prompt 組裝公式」

**Day 8–10 測試 3 個風格差異很大的 NPC**
- 陳姐（沉默的長者）✅ 已測
- 沈弈（尖刻悲觀的調查者）
- 蘇格蘭（精準冷靜的警探，會反問玩家）

**Day 11–12 設計「動態 Prompt 組裝公式」**

這是整個遊戲的引擎雛形。寫一個 Python 函式：

```python
def build_npc_prompt(
    npc_id: str,           # 例：'chen_jie'
    current_act: int,      # 1–10
    player_stats: dict,    # {EV, SRR, 好感度, 已得線索}
    npc_state: dict,       # {對玩家好感度, 已說過的線索}
    available_clues: list, # 這場可以放出的線索
    truth_string: str      # P3A1B2-7788-3-R7-D2-M1E3
) -> str:
    """組裝出當前狀態下，這個 NPC 應該收到的 System Prompt。"""
    ...
```

**這個函式 = 整個遊戲的腦袋。** Phase 0 只要寫出能跑的版本，不用完美。

**Day 13–14 寫 Phase 0 結案報告**
- Gemini vs Claude 對照表（哪個更會演？哪個更便宜？）
- Prompt 模板最終版
- 已知問題清單（例：「Gemini 偶爾會在第 8 輪後忘記人設」）
- **Go / No-Go 決策**：要不要往 Phase 1 走？

> 💡 **新手提醒**：Phase 0 最容易踩的坑是「寫了完美的 Prompt，但沒有測試紀錄」。請每次測試都複製對話貼進 markdown，**證據比感覺重要**。

## 🎲 Phase 3：隨機組合引擎（W7–W8，2 週）

> 目標：每次玩家「開新局」，遊戲會隨機抽出兇手 + 動機 + 關係 + 元素，並產出企劃書裡的自解碼字串 `P3A1B2-7788-3-R7-D2-M1E3`。

### W7 資料準備

- [ ] 把企劃書裡的所有 NPC、兇手、動機、元素**全部建表**到 Supabase
  - `npcs`（24 個一般 + 4 個固定 + 5 個事件相關 = 33 筆）
  - `suspects`（8 個兇手）
  - `motives`（4 個動機方向）
  - `relationships`（12 種失蹤者關係）
  - `truth_elements`（12 個輔助元素）
  - `compatibility_matrix`（哪些兇手適合哪些動機）
- [ ] **這一步可以請 AI 幫你把企劃書內容轉成 SQL INSERT** — 你只負責檢查正確性

### W8 遊戲引擎

- [ ] 寫 `lib/game-engine.ts`
  - `startNewGame(difficulty, route)` → 抽 1 個兇手 / 1 個動機 / 1 個關係 / 1–2 個元素
  - 檢查兼容性（例：兇手「白秋」必須搭方向 A 或 B）
  - 產出 `truth_string` 並存進 `game_sessions` 表
- [ ] **真相絕對不能傳回前端！** 真相只存在資料庫和 Edge Function 裡
- [ ] 寫一個 debug 頁面 `/debug/new-game`：點按鈕 → 顯示這局抽到的所有東西

### 完成標誌
連按 10 次「開新局」 → 看到 10 個不同的字串組合 → 都能合法解碼回真相。

## 🎬 Phase 4：第一條垂直切片（W9–W11，3 週）

> 目標：選一條最簡單的路線，把它做完整。玩家能從第一幕玩到一個結局，即使其他幕、其他路線都還沒做。

### 推薦的「最小可玩路線」
- **角色路線**：A（普通人類）— 系統較單純，不用做 EV/SRR
- **失蹤者關係**：R3（同事）— 情感重量適中，不會卡情緒
- **兇手**：莊河（退休刑警）— 對話線清楚，符合方向 B
- **長度**：第 1–3 幕 + 直接跳到第十幕的「沉默結局」

### W9 第 1–2 幕（場景與線索）
- [ ] 第一幕「普通的一天」：1 個場景 + 1 個 NPC（鄰居馬太太）+ 那則電視新聞
- [ ] 第二幕「空了」：玩家發現同事失聯 + 1 個 NPC（小游）
- [ ] 線索系統：用 Card 列表顯示玩家已得線索，可點開看詳情

### W10 第 3 幕 + 好感度系統
- [ ] 第三幕「你被懷疑了」：警局場景 + 蘇格蘭問話
- [ ] 蘇格蘭問話的「誠實 vs 說謊」選項
- [ ] 好感度系統：每個 NPC 一個數字，顯示在角色頁面（先用粗糙 UI）

### W11 跳關卡 + 第十幕沉默結局 + 5 個成就
- [ ] 在開發模式加一個「跳到第十幕」按鈕（正式版會拿掉）
- [ ] 沉默結局：純文字 + 打字機效果 + 第一幕電視新聞回呼
- [ ] 解鎖前 5 個成就：「相變世紀：Zero 通關」「沉默的共犯」「陳姐認可了你」「第幾個了」「便當裡的秘密」
- [ ] 簡單的成就列表頁面

### 完成標誌
你媽媽（或任何不懂程式的人）打開網址 → 不需要解釋就能玩 → 大概 30 分鐘玩到一個結局 → 看到成就被解鎖。

## 📊 風險與止損點

| 風險 | 何時會發現 | 止損方案 |
|---|---|---|
| Gemini 演不好 NPC | Phase 0 結束 | 改用 Claude Haiku，預算上升但品質有保障 |
| Supabase 免費額度爆掉 | Phase 4 朋友試玩時 | Vercel + Supabase 都升級到 $20/月就解決 |
| 自己卡住超過 3 天 | 任何階段 | 在這份計劃旁邊寫下「卡點日誌」，優先求助而不是硬撐 |
| 範圍蔓延（想加新功能） | 從 Phase 2 開始就會發生 | **任何不在本計劃裡的功能，全部丟到 `BACKLOG.md`，MVP 之後再看** |
| 失去動力 | 約莫 W5–W7 之間 | 拍一支 30 秒影片給朋友看現有進度，把回饋當燃料 |

## 🧭 給新手的五個提醒

1. **每天結束前 commit 一次。** 哪怕只改了一行字。Git 是你的時光機。
2. **Prompt 不是一次寫對的，是一次次測出來的。** 把每次調整都記在 `prompts/changelog.md`。
3. **遇到報錯先複製整段去問 AI，不要先猜原因。** Cursor / ChatGPT / Claude 都可以。
4. **API Key 永遠不放前端、不 push 上 GitHub。** 用 `.env.local` 並加到 `.gitignore`。
5. **每個 Phase 結束都寫一份 100 字小結。** 12 週後你會有一份完整的開發日誌，這對未來面試 / 求職 / 出版都非常有價值。

## 📅 12 週日曆速覽
